# ADR-0005: Firestoreによるジョブ永続化

**日付**: 2026-03-07
**ステータス**: 承認

## コンテキスト

Cloud Runはscale-to-zeroでインスタンスが破棄される。ジョブ履歴・レビューデータをインスタンスのメモリだけに保持すると、再起動で全消失する。

## 検討した選択肢

### 1. インメモリのみ
- ❌ インスタンス再起動で全データ消失
- ❌ 別デバイスから履歴が見えない

### 2. localStorage（クライアント側）
- ❌ ブラウザ依存（別デバイスで見えない）
- ❌ データが分散

### 3. Firestore（サーバー側）
- ✅ インスタンス非依存で永続化
- ✅ 無料枠で十分（読み取り50,000/日、書き込み20,000/日）
- ✅ asia-northeast1リージョンで低レイテンシ

## 決定

**選択肢3**: Firestore native mode。

### データモデル
```
scrape_jobs/{job_id}
  ├── reviews/{review_id}  (subcollection)
  └── logs/{auto_id}       (subcollection)
```

### 書き込み最適化
- レビューはバッチ書き込み（最大450件/バッチ、Firestore上限500のマージン）
- ログは `_progress_counter % 3 == 0` でスロットリング
- キャンセルチェックは `% 5 == 0`

## 結果

- サーバー再起動後も履歴・レビューが完全に残る
- 別デバイスからアクセスしても同じ履歴が見える
- コストは無料枠内（scale-to-zero = $0 when idle）
